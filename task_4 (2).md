# Тест-план для раздела «Корпоративное обучение» сайта «Школа 21»

## Тест-план

| Раздел          | Подраздел           | Содержание                                                                    |
|-----------------|---------------------|-------------------------------------------------------------------------------|
| **1. Введение** | 1.1. Цель документа | Настоящий документ описывает план тестирования раздела «Корпоративное обучение» сайта «Школа 21». Цель — обеспечение системного подхода к тестированию, определение scope, методологии, ресурсов и критериев качества для проверки функциональности, удобства использования и надежности данного раздела. |
|**2.Подход к тестированию**|2.1 Типы тестирования| Функциональное, UI-тестирование, Usability, кросс-браузерное и кроссплатформенное тестирование, тестирование производительности, безопасность (базовое).|
|  |2.2 Методологии|Ручное функциональное тестирование, техники эквивалентного разделения, анализа граничных значений, составление чек-листов и тест-кейсов. |
|**3. Окружение, инструменты и данные**|3.1 Бразуеры и устройства|**Десктоп:** Chrome, Firefox, Safari, Edge (последние версии)<br>**Мобильные:** iOS (Safari, Chrome), Android (Chrome, Samsung Internet)|
|  |3.2 Тестовые данные|Валидные/невалидные данные для формы, данные для проверки граничных значений. |
| **4. Расписание и метрики** | 4.1. Ориентировочное расписание | Подготовка: 1 день<br>Функциональное тестирование: 3 дня<br>Кросс-браузерное тестирование: 2 дня<br>Ре-тест: 2 дня<br>Отчет: 0.5 дня |
| | 4.2. Метрики | Общее количество тест-кейсов, % выполнения, количество дефектов, коэффициент дефектности. |
| **5. Роли и ответственности** | Команда проекта | **Тест-менеджер/Лид:** Утверждение плана, контроль, отчетность<br>**Тестировщики (2):** Выполнение тест-кейсов, документирование багов<br>**Разработчики:** Исправление дефектов<br>**Product Owner:** Предоставление требований, приемка результатов |
| **6. Артефакты тестирования** | Результаты работы | Тест-план, чек-листы и тест-кейсы, баги в системе трекинга, итоговый отчет о тестировании. |
